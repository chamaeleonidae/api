Changes are a record of every update made to your key models (Experiences).

- [Tours](apis/tours.md)
- [Mirosurveys](apis/surveys.md)
- [Launchers](apis/launchers.md)
- [Tooltips](apis/tooltips.md)
- [Steps](apis/steps.md)

---

Use the feed of changes to:

- Export a paper trail to know which changes made the biggest impact.
- Ensure that your most important Experiences stay static. Do this by listing all changes for a specific Experience and make sure the set of changes is empty
- Audit an issue with the delivery of an Experience.

## Schema

<ResponseField name="id" type="ID">
  The Chameleon ID
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  When this happened or when this was added to the Database
</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  The last time any property was updated
</ResponseField>

<ResponseField name="experience_id" type="ID">
  The ID of the parent model that this change represents. (a [Tour](apis/tour.md) when model is a [Step](apis/steps.md)
</ResponseField>

<ResponseField name="model_id" type="ID">
  The ID of the model that this change represents
</ResponseField>

<ResponseField name="model_type" type="string">
  The type of the model that this change represents
</ResponseField>

<ResponseField name="kind" type="string">
  The type of change this represents: One of: `create`, `update`, `destroy`, `raw`, `partial`, `related`, `revert`, or `apply`
</ResponseField>

<ResponseField name="path" type="string">
  They keypath this change represents
</ResponseField>

<ResponseField name="object" type="object">
  The underlying value of this change. It can be an object with scalar values (for a `kind=create` or `kind=destroy`) or values as a 2-item array of the [`before`, `after`] [diff](api/changes.md#schema-diff-item)
</ResponseField>

<ResponseField name="options" type="object">
  A hash of other information about this change. Values are a [diff](api/changes.md#schema-diff-item).
</ResponseField>

### Change "Diff item" Schema

Each value is a 2-item array `[BEFORE, AFTER]`. If the value changed from `"silver"` to `"gold"` then:

```
["silver", "gold"]
```

In the context of a Change `object`:

```json
{
  "plan": ["silver", "gold"],
  "monthly_spend": [99, 249]
}
```

## List Changes

List all Changes.

#### HTTP Request

```
GET https://api.chameleon.io/v3/edit/changes
GET https://api.chameleon.io/v3/edit/:model_kind/:model_id/changes
```

<ParamField query="limit" type="optional">
  Defaults to `50` with a maximum of `500`
</ParamField>

<ParamField query="before" type="optional">
  Used when paginating, use directly from the `cursor` object from the previous response
</ParamField>

<ParamField query="before" type="optional">
  Read as "created `before`" and can be given as a timestamp to get only `limit` items that were created before this time
</ParamField>

<ParamField query="after" type="optional">
  Read as "created `after`" and can be given as a timestamp or ID to get only `limit` items that were created after this time
</ParamField>

<ParamField query="filters" type="optional">
  An array of properties to filter `Changes` with; use this to fetch "all changes for a specific set of properties"
</ParamField>

<ParamField query="model_kind" type="optional">
  A `kind` of model to fetch changes for. One of `tour`, `survey`, `launcher`, or `tooltip`.
</ParamField>

<ParamField query="model_id" type="optional">
  The ID of the model to fetch changes for
</ParamField>

##### Examples

Only changes for the `6f3c4232c712de665632a6d5` Tour

```
GET https://api.chameleon.io/v3/edit/tours/6f3c4232c712de665632a6d5/changes
```

Only changes to the `published_at` property of any Experience

```
GET https://api.chameleon.io/v3/edit/changes?filter=published_at
```

# Summary of Changes

## Summary Schema

<ResponseField name="model_id" type="ID">
  The ID of the model that this change represents
</ResponseField>

<ResponseField name="path" type="string">
  They keypath this change represents
</ResponseField>

<ResponseField name="kind" type="string">
  The type of change this represents: One of: `create`, `update`, `destroy`
</ResponseField>

<ResponseField name="change_id" type="array">
  A 2-item array of the [`before`, `after`] [diff](api/changes.md#schema-diff-item) of Screen Capture ID of this change. Note some changes are no represented visually and will have null value for the `after` value
</ResponseField>

<ResponseField name="capture_id" type="array">
  A 2-item array of the [`before`, `after`] [diff](api/changes.md#schema-diff-item) of Screen Capture ID of this change. Note some changes are no represented visually and will have null value for the `after` value
</ResponseField>

<ResponseField name="object" type="object<ChangeUpdate>">
  A [ChangeUpdate](api/changes.md#schema-summary-change-update-item) of properties that have changed for this model
</ResponseField>

<ResponseField name="changes" type="array<ChangeUpdate>">
  An Array of [ChangeUpdate](api/changes.md#schema-summary-change-update-item) items representing changes to child associated models ([Steps](apis/steps.md), [Buttons](apis/buttons.md) etc.
</ResponseField>

### Change update object Schema

<ResponseField name="name" type="string">
  The name of the underlying property that changed
</ResponseField>

<ResponseField name="description" type="string">
  A longer description of the underlying property
</ResponseField>

<ResponseField name="value" type="object">
  The underlying value of this change. It can be an object with scalar values (for a `kind=create` or `kind=destroy`) or values as a 2-item array of the [`before`, `after`] [diff](api/changes.md#schema-diff-item)
</ResponseField>

##### Examples

```json
{
  "name": "Step created",
  "description": null,
  "value": {
    "id": "6f3c4232c712de665632a6d6",
    ...
  }
}
```

```json
{
  "name": "Body",
  "description": "The content of the Step",
  "value": ["New features are here!", "New features have arrived!"],
}
```


<ParamField path="model_kind" type="required">
